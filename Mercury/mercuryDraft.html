Building a
design system
for Mercury

This is a series of different projects were intended to explore and improve designs within the Mercury mobile ecosystem. The main goal is to build a cohesive design system that can work cross-functionally as well as streamline dependencies for engineers.


Selected Projects

UI Color Logic

Navigating the logic behind a complex color system for background "glows".

Cards

Exploring different layouts and structures to improve visual deliveries and language with cards.

Neural Networks

Exploring different layouts and structures to improve building neural networks.

Skeleton loader

Explore different designs and layout using various components for skeleton loading that can be used within Mercury.

Goal Statement

Cohesive visual style

The application needed to have a solid and cohesive style that would also be flexible enough for explorations and growth for future design sprints.

Artifical Intellegence

As an AI native application, we needed to design not just for our users, but also for our AI. A smart and clean system is essential to building a neural network that can learn effectively through user interactions.

Branding and customer acquisition

As a content driven application, it is essential to bring a cohesive design approach to bring trust and a stronger branding perspective throughout the application.
